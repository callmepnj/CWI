# CWI Unanswered Files - Image Sourcing & Rebuild Guide

**Last Updated:** May 24, 2026

## Overview

This guide provides detailed sourcing instructions for rebuilding all 18 case image folders with high-quality, authentic, case-specific images. Each image must be:
- ≥1200px wide (1600-3000px preferred)
- Freely licensed or public domain
- Case-specific (no generic/placeholder images)
- Non-repeated across folders
- Factually accurate captions

## Sourcing Strategy

### Primary Sources (in order of preference)
1. **Wikimedia Commons** - Free, CC-licensed images, search by case keywords
2. **Official Government Sources** - Ministry of Earth Sciences, Forest Department, Election Commission
3. **News Archives** - Reuters, AP, BBC, Al Jazeera, The Hindu, Indian Express, Scroll, The Wire
4. **Human Rights Organizations** - HRW, Amnesty International, Article 14, Frontline
5. **Specialized Resources** - Land Conflict Watch, local government sites

### Copyright Rules
- ✅ Use CC BY-SA 4.0 or CC0 freely
- ✅ Use public domain images
- ✅ Use official government/public domain materials
- ⚠️ Reference news articles but download only legally available/licensed versions
- ❌ DO NOT scrape/illegally download copyrighted images
- ❌ DO NOT use AI-generated fake "news" images

## Image Sourcing Workflow

### Step 1: Search Wikimedia Commons
```
1. Go to commons.wikimedia.org
2. Search by case keywords (e.g., "Manipur" OR "Hathras")
3. Filter by license: CC BY-SA 4.0, CC0, Public Domain
4. Check image resolution (min 1200px wide)
5. Verify relevance to case
6. Download full-size version
7. Save with clean filename: image-01.jpg, image-02.jpg, etc.
```

### Step 2: Document Source
For each image, record:
- Filename (hero.jpg or image-XX.jpg)
- Title (what it shows)
- Caption (factual description)
- Source (Wikimedia Commons, news outlet, etc.)
- Source URL (direct link)
- Photographer
- License (CC BY-SA 4.0, etc.)
- Date photographed
- Usage note

### Step 3: Validate Before Upload
- Check resolution (≥1200px wide)
- Verify license
- Confirm case-specific relevance
- Ensure non-graphic
- Check for duplicates

### Step 4: Upload & Update Metadata
1. Place image in correct case folder
2. Update metadata.json with details
3. Run validation script
4. Test on website

## Batch Sourcing Tips

### For Multiple Images
1. **Wikimedia Advanced Search:**
   - Use category filter (e.g., "Manipur")
   - Use date range filter
   - Use license filter

2. **News Archive Search:**
   - Reuters: reuters.com/pictures
   - AP: apimages.com
   - BBC: bbc.com/news/in_pictures
   - Check license terms before downloading

3. **Verification Websites:**
   - TinEye for reverse image search (check for duplicates)
   - Exiftool for metadata verification
   - FastStone Image Viewer for batch dimension checking

## Red Flags (Images to AVOID)

❌ **DO NOT USE:**
- Pixelated or compressed thumbnails
- Stretched or distorted images
- Screenshots with UI elements
- Random Northeast/Southern India images (wrong case)
- Repeated images across cases
- Generic protest stock photos
- Dark backgrounds repeated everywhere
- Fake AI-generated "news" images
- Copyrighted images without clear licensing
- Unrelated images (e.g., tourism photos for activism case)
- Graphic violence or exploitative images
